================================================================================
            .NET 애플리케이션 리버스 엔지니어링 분석 보고서
================================================================================

[질문]
빌드한 프로그램을 다른 사람이 디코딩해서 보거나 수정할 수 있는가?

[결론]
예, 다른 사람이 소스 코드를 거의 원본 수준으로 복원할 수 있습니다.


================================================================================
1. 왜 그런가? - .NET 컴파일 특성
================================================================================

.NET 프로그램의 컴파일 과정:

  C# 소스코드  -->  IL (Intermediate Language)  -->  CLR에서 실행
                          ^
                          |
                   이 단계에서 쉽게 역변환 가능

.NET은 Java와 마찬가지로 중간 언어(IL)로 컴파일됩니다.
이 IL 코드는 원본 소스코드와 거의 1:1로 대응되어 쉽게 복원됩니다.


================================================================================
2. 현재 프로젝트 설정 분석
================================================================================

현재 csproj 설정:

  설정 항목                        값          보호 효과
  -------------------------------------------------------
  PublishSingleFile               true        번들링만 (보호 X)
  SelfContained                   true        배포 편의 (보호 X)
  EnableCompressionInSingleFile   true        압축만 (보호 X)
  난독화 (Obfuscation)            없음        무방비 상태
  Native AOT                      없음        무방비 상태

결론: 현재 설정으로는 코드 보호가 전혀 되지 않습니다.


================================================================================
3. 디컴파일 도구 (누구나 무료로 사용 가능)
================================================================================

1) dnSpy
   - 가장 강력한 도구
   - 코드 보기 + 수정 후 재컴파일까지 가능
   - GitHub에서 무료 다운로드

2) ILSpy
   - 오픈소스, 가장 널리 사용됨
   - Visual Studio 확장도 제공

3) dotPeek (JetBrains)
   - 무료, 고품질 디컴파일러
   - JetBrains에서 제공

4) JustDecompile (Telerik)
   - 무료, 사용하기 쉬움


================================================================================
4. 디컴파일 시 노출되는 정보
================================================================================

[보이는 것]
- 모든 클래스, 메서드, 변수명
- 문자열 상수 (API URL, 에러 메시지, 만료일 등)
- 비즈니스 로직 전체
- 알고리즘 및 데이터 처리 방식

[보이지 않는 것]
- 주석 (컴파일 시 제거됨)
- 일부 지역 변수명 (최적화된 경우)

실질적으로 원본 코드의 90% 이상이 복원됩니다.

5. 보호 옵션
================================================================================

[옵션 1] 난독화 (Obfuscation) - 가장 현실적인 방법

  도구명              비용      효과
  -----------------------------------------
  ConfuserEx          무료      기본적인 난독화
  Dotfuscator         무료/유료 문자열 암호화 포함
  Eazfuscator.NET     유료      강력한 보호
  .NET Reactor        유료      안티-디버깅 포함

  난독화 효과:
  - 클래스/메서드/변수명을 의미없는 문자로 변경
    예: GetUserData() --> a0x9f3()
  - 문자열 암호화 (API URL 등 숨김)
  - 제어 흐름 난독화 (코드 흐름 복잡하게)
  - 안티-디버깅 (디버거 탐지 시 종료)


[옵션 2] Native AOT 컴파일 (.NET 8 지원)

  <PublishAot>true</PublishAot>

  - IL 대신 네이티브 기계어로 직접 컴파일
  - 디컴파일 난이도 대폭 상승
  - 단점: WinForms는 현재 AOT 지원이 제한적


[옵션 3] 하드웨어 바인딩

  - 라이선스 키 + 하드웨어 ID 검증
  - 특정 PC에서만 실행되도록 제한
  - 복제 방지 효과

6. 현실적 권장사항
================================================================================

[보호 수준별 추천]

  수준      방법                  비용    효과
  -----------------------------------------------
  기본      ConfuserEx            무료    일반 사용자 방어
  중간      Dotfuscator Community 무료    문자열 암호화 추가
  높음      상용 난독화 도구      유료    전문가도 분석 어려움


[상황별 추천]

  1. 무료 배포 + 개인 사용 목적
     --> 기본 난독화 (ConfuserEx)로 충분
     --> 호기심 있는 일반 사용자 방어 가능

  2. 상업적 가치가 있는 프로그램
     --> 상용 난독화 도구 권장
     --> .NET Reactor 또는 Eazfuscator.NET

  3. 민감한 알고리즘/API 키 포함
     --> 서버 사이드로 이동 권장
     --> 클라이언트에는 최소한의 로직만


================================================================================
7. 중요한 현실
================================================================================

완벽한 보호는 불가능합니다.

충분한 시간과 기술이 있는 사람은 어떤 보호도 우회할 수 있습니다.
난독화의 목적은 "불가능하게 만드는 것"이 아니라
"충분히 어렵게 만들어 포기하게 하는 것"입니다.

비용 대비 효과를 고려하여 적절한 수준의 보호를 선택하세요.


================================================================================
8. ConfuserEx 적용 방법 (무료)
================================================================================

1) ConfuserEx 다운로드
   https://github.com/mkaring/ConfuserEx/releases

2) 프로젝트 설정
   - ConfuserEx.exe 실행
   - 빌드된 exe 파일 추가
   - 보호 규칙 설정 (anti-tamper, constants, ctrl-flow 등)

3) 난독화 실행
   - Protect 버튼 클릭
   - 출력 폴더에 난독화된 exe 생성

난독화 적용을 원하시면 말씀해주세요.
상세한 설정 방법을 안내해드리겠습니다.
